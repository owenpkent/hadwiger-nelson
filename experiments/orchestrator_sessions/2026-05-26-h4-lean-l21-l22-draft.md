## Definitions formalized (10)

In `Bridges.lean`:

1. `liftLeft (H_1 : SimpleGraph V_1) : SimpleGraph (V_1 + V_2)`. Lifts `H_1` to the disjoint-union type: adjacent only on the `inl` side.
2. `liftRight (H_2 : SimpleGraph V_2) : SimpleGraph (V_1 + V_2)`. Symmetric lift on the `inr` side.
3. `bridgeSetGraph (B : Set (V_1 x V_2)) : SimpleGraph (V_1 + V_2)`. Edges only between `inl u` and `inr v` for `(u, v) in B`.
4. `bridgeGraph (H_1 : SimpleGraph V_1) (H_2 : SimpleGraph V_2) (B : Set (V_1 x V_2)) : SimpleGraph (V_1 + V_2)`. The supremum `liftLeft H_1 sup liftRight H_2 sup bridgeSetGraph B`. This is the central construct.

In `L21CoveringLemma.lean`:

5. `combinedColoring`: given proper 4-colorings of the two halves with the bridge non-collision property, build a proper 4-coloring of `bridgeGraph` via `Sum.elim`.
6. `leftRestrict`: project a coloring of `bridgeGraph` to a coloring of `H_1`.
7. `rightRestrict`: project a coloring of `bridgeGraph` to a coloring of `H_2`.

In `L22ListColoring.lean`:

8. `ListColoring G L`: a structure packaging a `G.Coloring (Fin 4)` plus the membership condition `c v in L v` for all `v`.
9. `ListColorable G L : Prop`: `Nonempty (ListColoring G L)`.
10. `forbiddenColors c_1 B v : Set (Fin 4)` and `allowedColors c_1 B v : Set (Fin 4)`. The `F(v)` and `L(v)` of LEARNINGS L22.

## Theorems proved (8, all without `sorry`)

In `L21CoveringLemma.lean`:

1. `bridge_noncollision`: the bridge non-collision property is preserved when splitting a combined coloring.

2. `bridgeGraph_colorable_four_iff` (L21, 4-colorable form):
```
theorem bridgeGraph_colorable_four_iff (H_1 : SimpleGraph V_1) (H_2 : SimpleGraph V_2)
    (B : Set (V_1 x V_2)) :
    (bridgeGraph H_1 H_2 B).Colorable 4 <->
      exists (c_1 : H_1.Coloring (Fin 4)) (c_2 : H_2.Coloring (Fin 4)),
        forall p in B, c_1 p.1 != c_2 p.2
```

3. `bridgeGraph_not_colorable_four_iff` (L21, chi >= 5 form, contrapositive):
```
theorem bridgeGraph_not_colorable_four_iff :
    not (bridgeGraph H_1 H_2 B).Colorable 4 <->
      forall (c_1 : H_1.Coloring (Fin 4)) (c_2 : H_2.Coloring (Fin 4)),
        exists p in B, c_1 p.1 = c_2 p.2
```

4. `bridgeGraph_five_chromatic_of_covering`: the implication form. If every coloring pair is hit, chi(G) > 4.

In `L22ListColoring.lean`:

5. `noncollision_iff_list_membership` (the bookkeeping lemma): for a fixed `c_1`, a coloring `c_2` of `H_2` satisfies the L21 non-collision condition iff `c_2 v in allowedColors c_1 B v` for every `v`.

6. `bridgeGraph_colorable_four_iff_list` (L22, chi <= 4 form):
```
theorem bridgeGraph_colorable_four_iff_list :
    (bridgeGraph H_1 H_2 B).Colorable 4 <->
      exists c_1 : H_1.Coloring (Fin 4),
        ListColorable H_2 (allowedColors c_1 B)
```

7. `bridgeGraph_not_colorable_four_iff_list` (L22, chi >= 5 form):
```
theorem bridgeGraph_not_colorable_four_iff_list :
    not (bridgeGraph H_1 H_2 B).Colorable 4 <->
      forall c_1 : H_1.Coloring (Fin 4),
        not ListColorable H_2 (allowedColors c_1 B)
```

8. `L21_iff_L22`: the L21 covering condition (every coloring pair hit) is EQUIVALENT to the L22 universal-list-uncolorable condition. Proved by chaining the two `not ... iff ...` characterizations through the negation of `Colorable 4`.

## Design notes

**Disjoint vertex types via `Sum`**. The LEARNINGS statement says "$H_1, H_2$ on disjoint vertex sets". Two options:
  (a) keep a single vertex type `V` and require `V_1, V_2 subseteq V` with `V_1 cap V_2 = empty` as a hypothesis, or
  (b) use the disjoint-union type `V_1 + V_2` directly.

We chose (b). Pros: disjointness is `Sum.inl_ne_inr` automatically, no boundary-condition lemmas. Cons: any concrete application (e.g., taking `H_1, H_2 :=` two copies of the Moser spindle) requires a small `Sum.inl`/`Sum.inr` wrapping at the call site.

For the L23-L24 lift to three halves (`triple_chi_six_iff` from the Adversary/Synthesizer drafts), we'd use `V_1 + V_2 + V_3`, and the same machinery generalizes cleanly via `Sum.elim` for the combined coloring.

**`ListColoring` as a structure, not a predicate**. We defined `ListColoring G L` as a structure (data: the coloring + the membership proof) and `ListColorable G L := Nonempty (ListColoring G L)` as the prop. This matches Mathlib's `Coloring G alpha` and `Colorable G n` design. If/when Mathlib adds a list-coloring API of its own, we can swap to that with minimal friction.

**Allowed colors as `Set (Fin 4)`, not `Finset`**. The L22 statement is purely propositional on `Fin 4`, so `Set` is the natural ambient type. If we later want to count or enumerate `allowedColors`, a `Finset` version is a one-line construction (`Finset.univ.filter`).

## Future formalization directions

1. **Concrete instantiation: the Moser x Moser 14-bridge graph**. The natural next step is to define `mossquareGraph := bridgeGraph moserSpindle moserSpindle B_14` where `B_14` is the LEARNINGS L21 explicit 14-bridge set, and prove `mossquareGraph.chromaticNumber = 5` via either:
   - a `native_decide` / SAT certificate route (the cleanest for finite cases), or
   - a direct L22 application: enumerate the 16 = 4! / |orbit| canonical colorings `c_1` of `moserSpindle` (or use the fact that all 16 are `S_4`-equivalent), and prove each gives a list-uncolorable instance on the second copy of `moserSpindle`. The "uncolorable" instance at the canonical `c_1 = (0,1,2,0,1,2,3)` reduces (by L22) to the L21 14-bridge no-K_4 result.

2. **L23-L24 lift to three halves (chi >= 6 form)**. The triple-coupling theorem from [`2026-05-26-l24-3way-draft.md`](2026-05-26-l24-3way-draft.md) generalizes L21 / L22 to three 5-chromatic halves. The Lean lift uses `V_1 + V_2 + V_3` and a hypergraph of three pairwise bridge sets. The proof structure is the same: `Sum.elim`-style construction of the combined coloring, restriction lemmas in three directions, and the list-coloring bookkeeping lemma applied symmetrically.

3. **Connect to `planeUnitDistanceGraph` via the Moser-bridge homomorphism**. The L21 / L22 results are graph-theoretic and abstract. Connecting them to `chromaticNumberOfPlane` requires:
   - prove that the abstract `bridgeGraph H_1 H_2 B` admits a graph homomorphism `bridgeGraph -> planeUnitDistanceGraph` when `H_1, H_2` are realized as UDGs in `R^2` and `B` is a set of unit-distance bridges. (Realizability is a separate, geometric question; LEARNINGS L21 notes the 14-bridge Moser x Moser is presumably NOT UDG-realizable in `R^2`).
   - infer `4 <= chromaticNumberOfPlane <= 5` is already proved; `5 <= chromaticNumberOfPlane` is the de Grey 2018 result and would require the SAT-certificate / DRAT-checker approach for the 1581-vertex graph.

4. **DRAT-checker integration**. A natural follow-on H-task: implement (or import) a minimal LRAT/DRAT checker in Lean and use it to verify the de Grey 1585 UNSAT certificate. This would give a formal proof of `chromaticNumberOfPlane >= 5`. Heule has done this in Coq; the Lean port is an open formal-math project. Outside the scope of this session.
