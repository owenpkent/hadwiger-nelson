The headline finding is **angle 6**: L20's implication 5 ("a third 4-chromatic component coupled by bridges does not force chi=6 because the third color class can be reused") is **wrong** in general. It holds for some bridge layouts (asymmetric, chain, permuted) but FAILS dramatically when the bridge layouts are aligned on the same vertex orbits across all three pairs.

## Detailed numerical findings

### Angle 1: triviality boundary (combinatorial)

For H1 = H2 = Moser spindle (abstract), exhaustive over all 7*7 = 49 single-bridge configurations:
- 0 of 49 single-bridge placements force chi >= 5.
- The "trivial K_5" via H2-neighborhood does NOT trigger from a single bridge because Moser has no K_4 (clique number = 3).

For H1 = H2 = K_4 (clique number = 4):
- 1 bridge: chi = 4.
- 4 bridges forming K_{1,4} (star from H1[0] to all of H2): chi = 5 (creates K_5).

This sets the triviality threshold: chi-5 forcing requires AT LEAST 4 bridges, and "many bridges from one vertex" is the trivial route via K_5. BUILDER's minimum for Moser-Moser is 6 (greater than the trivial K_5 size since Moser has only K_3 cliques).

### Angle 2: Q^2 detector (Woodall control)

Sampled Q^2 unit-distance graph: 76 vertices generated from Pythagorean triple-derived unit vectors + their translates. SAT-computed chi = 2 (matches Woodall). 

Implication: L20's halves-plus-bridges framework intrinsically requires irrational coordinates because no 4-chromatic UDG embeds in Q^2. The framework is therefore NOT a structural property of unit-distance graphs in general; it depends on the existence of 4-chromatic UDGs (Moser spindle) which requires R \ Q. This is consistent with all known chi>=5 constructions using algebraic number fields.

### Angle 3: L^infty detector (Chilakamarri control)

King graph 5x5: chi = 4 (matches Chilakamarri).

Pairwise chi=5 trivialization test: Two K_4s + 4-star bridge = K_5 = chi 5. This is trivially achievable as a graph structure (no L^infty-UDG embedding required at this size; the K_5 fits if 5 vertices at pairwise L^infty distance 1 existed, which they do not in L^infty). So the abstract halves-plus-bridges framework gives chi=5 via the K_5 route, but L^infty's chi=4 cap prevents this from being realized as a UDG.

Interpretation: the L20 framework as a *combinatorial* claim is normally trivializable by K_5 forcing; what makes it interesting for HN is that the bridges must be UNIT-DISTANCE edges in R^2, which adds geometric constraints that prevent the K_5 route.

For all 49 cross-bridges: chi = 8 = chi(H1) + chi(H2). The framework saturates at chi(H1)+chi(H2) for dense bridges.

### Angle 5: color-class collision lower bound

Moser spindle: 384 proper 4-colorings (16 canonical mod S_4 -- matches BUILDER's n_C2 = 384, n_C1 = 16).

f(u, v) := P[c1(u) = c2(v) | random pair of 4-colorings c1, c2] = **0.25 for ALL 49 (u, v) pairs**.

This uniformity is striking: every Moser vertex has the same color-class distribution (each color equally likely). So the collision probability is independent of the bridge endpoint choice. This matches BUILDER's finding rho_min = rho_mean = rho_max = 0.25.

Union bound: need sum_{(u,v) in B} f(u, v) >= 1, so |B| >= 4.

BUILDER's minimal |B| = 6 exceeds this bound by 50%. The union bound is therefore loose for the Moser case because the killed sets K_{u, v} of bridges overlap substantially.

Implication: union bound is a weak lower bound. The true minimum |B| is determined by the *combinatorial structure* of which colorings each bridge kills, not by simple probabilities.

**Strong refutation of L20 implication 5**: the claim "third color class can be reused" assumed the bridge configurations are independent. When B12, B13, B23 are *aligned* on the same vertex orbits across all three spindles, the coupling propagates. We find chi >= 6 in many configurations, including aligned K_{2,2} which only has 4 bridges per pair and does NOT even force pairwise chi=5.

**The trivial K_6 path (combinatorial)**: aligned K_{2,2} bridges between two-vertex K_2-cliques in each spindle creates a literal K_6 subgraph:
- Pick K_2 in H1 (say {0, 1}; verified 0-1 is a Moser edge).
- Pick K_2 in H2 ({0, 1}) and H3 ({0, 1}) similarly.
- All K_{2,2} bridges in each pair: {H1[0,1]} x {H2[0,1]} x {H3[0,1]} forms K_6 (15 edges among 6 vertices, all present).
- chi = 6 trivially via K_6.

Verified: H1 = H2 = H3 = Moser, B = {(0,0),(0,1),(1,0),(1,1)} aligned: produces K_6 on the 6 vertices {0, 1, 7, 8, 14, 15}.

**Critical UDG implication**: this K_6 trivial route is BLOCKED for UDG in R^2. The unit-distance clique number in R^2 is 3 (equilateral triangle); K_4 already requires R^3 (regular tetrahedron). So the aligned K_{2,2} construction does NOT lift to a chi=6 UDG. This means the combinatorial chi=6 found here is NOT a chi=6 UDG witness.

**Refined L20 implication 5 (corrected)**: the third color class CAN be reused under any of:
- The bridge sets B12, B13, B23 are vertex-disjoint or only share via permutations.
- The bridge sets do not align "K_2 in H1 -> K_2 in H2" creating K_4-type cross structures.

The third color class CANNOT be reused (and chi >= 6 follows) when:
- The bridges align K_r in H1 with K_r in H2 with K_r in H3 for r >= 2, creating K_{r}*r = K_{3r} cross structures.

For HN UDG: the constraint that K_4 is not a UDG in R^2 means r <= 2 only allows K_4 cross, not K_6. Hence the "trivial" route to chi=6 via aligned K_{2,2} is BLOCKED. A 3-way chi=6 UDG needs a STRUCTURAL force beyond aligned K_2 bridges, exactly as L20 conjectured but with a different obstruction explanation.

**Practical combinatorial chi>=6 construction (not a UDG)**: three Moser spindles + aligned K_{2,2} bridges between {0,1} in each spindle = 21 vertices, 33 + 12 = 45 edges, chi = 6. Even simpler: three triangles K_3 + K_{2,2} bridges between {0,1} in each = 9 vertices + lots of K_6 = chi = 6. None of these are UDGs.

## Implications for BUILDER's L21 entry

1. **L20 implication 5 needs major revision**. The claim "third color class can be reused" is correct for *some* bridge configurations (permuted, asymmetric, chain), but FALSE for aligned configurations. In particular, aligned K_{2,2} bridges between K_2-cliques in three spindles create a literal K_6, forcing chi=6 trivially. Aligned K_{2,3} bridges (i.e., B6 from BUILDER on the same vertices in all three pairs) push to chi=7.

2. **The chi=6 forcing has a trivial combinatorial route**. The 3-way aligned K_{2,2} K_6 construction is the analog of "K_4 + K_4 + 4-star = K_5" at the chi=6 level. Both are TRIVIAL combinatorial chi>=k+1 forcings via K_{k+1} subgraphs. Neither lifts to a UDG in R^2:
   - K_5 in UDG: requires 5 mutually-unit-distance points = impossible in R^2.
   - K_6 in UDG: requires 6 mutually-unit-distance points = impossible in R^2 (max clique is 3).

3. **The non-trivial chi=6 route is still open**. To force chi=6 via UDG, we need a NON-K_6 combinatorial obstruction. L20's framework hints at this but doesn't characterize when. Our angle 6 data shows that:
   - Aligned K_{2,3} 3-way: chi=7 (also blocked by UDG K_6 constraint, since K_{2,3} aligned on edges still creates K_6).
   - Permuted / chain / asymmetric: chi=5 (consistent with L20 implication 5).

   So the "third color class can be reused" is correct precisely when the bridge configuration AVOIDS aligned K_{2,2} on edges. The chi=6 question is: can a UDG-realizable 3-way bridge configuration force chi=6 without creating K_6? Open.

4. **The union bound is loose**. f(u,v) = 0.25 uniform on Moser gives lower bound |B| >= 4, but the actual minimum is 6. The combinatorial structure of kill-set overlaps determines the gap. BUILDER's set-cover formulation correctly captures this.

5. **Q^2 detector**: confirms L20 framework requires irrational coordinates. Not a refutation, but the right structural attribution.

6. **L^infty detector**: trivializes the abstract halves-plus-bridges framework via K_5 forcing. The framework's *non-trivial* content for HN comes from the unit-distance constraint preventing K_5. Without this constraint, halves-plus-bridges is just "make a K_{k+1}".

## Counterexamples and witnesses (explicit)

**Strongest adversarial witness (aligned K_{2,2} 3-way Moser, chi=6 via K_6)**:
- H1 = H2 = H3 = Moser spindle (7 vertices, 11 edges each)
- B12 = B13 = B23 = [(0,0), (0,1), (1,0), (1,1)] (4 bridges each, K_{2,2} on K_2 cliques)
- Combined graph: 21 vertices, 45 edges, chi = 6.
- The 6 vertices {H1[0], H1[1], H2[0], H2[1], H3[0], H3[1]} form a literal K_6 in the union graph (15 edges among them, all present). Hence chi >= 6 trivially.
- Pairwise chi (any two spindles): only 4. So this construction forces chi=6 WITHOUT ever achieving pairwise chi=5. Total refutation of L20 implication 5's "pairwise-5 needed for chi=6".

**Counterexample to L20 implication 5 (3-way naive bound, aligned K_{2,3})**:
- H1 = H2 = H3 = Moser spindle
- B12 = B13 = B23 = [(0,0), (0,1), (0,2), (1,0), (1,1), (1,2)]
- Combined: 21 vertices, 51 edges, chi = 7.

**Witness for chi=6 from 3-way K_4 coupling**:
- H1 = H2 = H3 = K_4 (vertices {0,1,2,3} for each)
- B12 = B13 = B23 = [(0,0), (0,1), (0,2), (0,3)] (K_{1,4} stars)
- Combined: 12 vertices, 30 edges, chi = 6. This is K_4 * K_4 * K_4 sharing apex = K_6 via apex.

**Triviality witness (single-bridge)**:
- H1 = H2 = K_4, B = {(0, 0)}: chi = 4. Even single bridge between cliques does not force chi=5.

**Triviality witness (K_{1,4} on K_4-K_4)**:
- H1 = H2 = K_4, B = {(0,0), (0,1), (0,2), (0,3)}: chi = 5 (the K_5 trivial route).

## Caveats

- All angle-6 results are *combinatorial*, not yet *unit-distance*. The question of whether the aligned 3-way coupling can be realized as a UDG in R^2 is open and is the BUILDER's next investigation. ADVERSARY's role here is to flag that L20 implication 5 is wrong, not to construct the UDG realization.
- The Moser triple aligned configuration gives chi=7, not chi=6. To find the *minimum* chi=6 3-way configuration on Moser may require fewer / different bridges. ADVERSARY did not exhaustively minimize for chi=6; only verified the aligned chi=7 case and the K_4-triple chi=6 case.

## Verdict summary

PASS for angles 1, 2, 5 (no triviality at single-bridge, controls hold, union bound discipline OK).

PARTIAL for angles 3, 4 (frameworks have specific abstract trivializations; UDG constraints rescue them).

FAIL for L20 implication 5 (angle 6): the naive 3-way coupling claim is false in two ways:
- Aligned K_{2,2} 3-way produces a K_6 subgraph and forces chi=6 with only 4 bridges per pair and pairwise chi=4 only.
- Aligned K_{2,3} 3-way (BUILDER's minimal pairwise-5 set replicated) produces chi=7.

BUT: both routes to chi>=6 require K_5 / K_6 subgraphs that are *blocked* in R^2 by the unit-distance clique bound (max UDG clique in R^2 = K_3). So the abstract chi=6 forcings DO NOT lift to UDG witnesses.

**Net for L21**: L20 implication 5 must be rewritten as a conditional. The third color class can be reused IFF the bridge configuration avoids forming K_{r,r,r} (or K_{r}*r) cross-structures aligned with K_r cliques in each spindle for r >= 2. The pure combinatorial "halves + bridges" framework has trivial chi=6 forcings that UDG geometry rules out. The chi=6 UDG question is therefore: does a NON-K_6 combinatorial obstruction exist within the UDG-realizable bridge configurations?

This is a sharper open question than L20 left it: BUILDER's next goal should be characterizing the UDG-realizable 3-way bridge configurations and SAT-searching them for chi=6 obstructions that don't go through K_5 / K_6.
